# Автосбор лидов (Sprint 1) — краткий гайд

## Что уже автоматизировано

- Сбор тендерных лидов из:
  - `TED` (EU notices API)
  - `Cyprus eProcurement` (opened tenders list)
  - `Zakupki.gov.ru` (RU, extended search parsing)
  - `Goszakup.gov.kz` (KZ, OWS API)
  - `Icetrade.by` (BY, tenders list parsing)
- Раздельная логика по гео-бакету:
  - `CYPRUS_EN`: `TED (CYP)` + `Cyprus eProcurement`
  - `RU_CIS`: `TED` + RU/CIS источники (`Zakupki`, `Goszakup`, `Icetrade`), Cyprus источник отключается
- Нормализация в единый формат лида.
- Дедупликация по `source_uid` (например, `ted:749524-2023`).
- Базовый lead scoring (бюджет, дедлайн, CPV, повторяемость заказчика, источник).
- Приоритет в скоринге для лидов с найденным email (чтобы выше были карточки, готовые к рассылке).
- Импорт в карточки внутри `vadim-filter-tool.html`.
- Кэш лидов в helper:
  - `helper/.cache/lead_cache_v1.json`
- В статусе автосбора видно `email_ready` (сколько карточек сразу пригодны для Email).

## Что осталось ручным

- Проверка топ-лидов человеком (рекомендация: минимум топ-10 по скорингу в день).
- Финальное решение по запуску коммуникации (email/звонок/мессенджеры).

## Как пользоваться в интерфейсе

1. Запустите helper (`START_MAC.command` или `START_WIN.bat`).
2. Откройте вкладку `Импорт`.
3. Для ручного режима используйте вкладку `Ручной импорт`.
4. Для автоматического режима откройте вкладку `Автопоиск`.
5. В блоке `Автосбор лидов (пилот)`:
   - выберите источники (`TED`, `Cyprus eProcurement`, `Zakupki`, `Goszakup`, `Icetrade`),
   - выберите `Гео-база` (`CYPRUS_EN` для пилота),
   - включите `Глубокий парсинг detail-страниц` (если нужен добор email/телефонов),
   - задайте `Детальных карточек на источник` (баланс скорость/качество),
   - при необходимости вставьте `Goszakup API token`,
   - задайте лимит и число страниц Cyprus eProcurement,
   - нажмите `Собрать и импортировать`.
6. Перейдите в `Карточки` и отфильтруйте:
   - по `Workspace` (`manual` / `auto`),
   - по `Гео-база`,
   - по `Источник`,
   - по `Когорте контакта` (`Email-ready`, `WA/TG без Email`, `Нет контактов`).
7. В карточках/таблице есть колонка/поле `Ссылка` для перехода к источнику.
8. Для рассылки используйте кнопку `Выбрать Email-ready`, для ручного контакта — `Выбрать без Email (WA/TG)`.
9. Экспорт `CSV/Excel` теперь включает поля `Когорта контакта` и `Каналы контакта`.

## Режим авто-таймера

- В блоке автосбора есть переключатель:
  - `Автосбор по таймеру (работает пока открыта эта вкладка)`.
- Интервал задается в минутах.
- Важно: это клиентский таймер во вкладке браузера, а не отдельный системный cron.

## API helper (локальные ручки)

- `GET /leads/health`
- `GET /leads/list`
- `POST /leads/collect`

Все ручки ожидают заголовок:
- `X-SB-Client: vadim-filter-tool`

## Ограничения текущей версии

- Для `Goszakup.gov.kz` может потребоваться API-token (`GOSZAKUP_API_TOKEN`), иначе источник вернет warning и пустой fetch.
- Часть RU/CIS источников может быть ограничена по доступу из некоторых регионов/сетей (таймауты не ломают общий автосбор, а идут в warnings/source_reports).
- `Глубокий парсинг detail-страниц` увеличивает качество контактов, но может заметно замедлить запуск.
- Для Cyprus eProcurement используется список opened tenders (без полного карточного парсинга деталей, т.к. detail-страницы требуют авторизацию).
- Скоринг базовый и будет уточняться на реальных сделках.
