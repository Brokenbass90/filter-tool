# Журнал плана Strong Bridge Tool

Обновлено: 2026-03-09

## Цель

Довести продукт до состояния, где он:
- стабильно запускается на macOS и Windows;
- предсказуемо собирает ручные и авто-лиды;
- помогает доводить лиды до контакта и сделки, а не только хранить карточки.

## Приоритеты

### P0. Стабильность запуска и диагностика
- [x] Windows-бандл с актуальной версией собран.
- [x] `START_WIN.bat` пишет отдельный лог запуска.
- [x] helper теперь пишет отдельный runtime-лог в `logs\helper_*.log`.
- [x] в интерфейсе есть кнопка скачивания JSON-диагностики.
- [ ] добавить быстрый self-check сценарий для партнера: старт, `/health`, импорт, тест-письмо.

### P1. Предсказуемый автопоиск
- [x] отдельная страница автопоиска.
- [x] диагностика источников и сброс автопоиска.
- [x] сделать еще более явный статус по каждому источнику: `ok / token required / blocked by network / 0 results`.
- [x] отдельно подсвечивать, какие лиды пришли с контактами, а какие без.

### P2. Контактная пригодность карточек
- [x] email / WhatsApp / Telegram кнопки.
- [x] авто и ручной workspace уже разведены.
- [x] ввести когорты контактов: `email-ready`, `phone-only`, `tg/wa-only`, `no-contacts`.
- [x] добавить фильтры и экспорт по этим когортам.
- [x] добавить быстрый перевод лидов без email в ручную очередь контакта.

### P3. Продажи и воронка
- [ ] шаблоны писем по гео и языку: `RU_CIS`, `EN_CYPRUS`.
- [ ] follow-up статусы и менеджерские задачи.
- [ ] простая отчетность: `найдено -> готово к контакту -> отправлено -> ответили -> сделка`.

## Рабочий порядок

1. Сначала снижаем риск техподдержки и проблем запуска.
2. Потом делаем автопоиск понятным и управляемым.
3. Потом усиливаем карточки и каналы контакта.
4. Только после этого расширяем CRM-функции и growth-фичи.
