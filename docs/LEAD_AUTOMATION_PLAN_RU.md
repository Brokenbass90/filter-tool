## 1) Цель
Сделать так, чтобы база в программе пополнялась автоматически каждый день, а человек вручную проверял только приоритетные лиды.

## 2) Что делает автоматизация, что делает человек
Автоматизация:
1. Забирает новые тендеры/компании из выбранных источников.
2. Нормализует данные в единый формат карточки.
3. Ставит приоритет (lead score) по правилам.
4. Кладет лиды в базу и ставит задачи follow-up.
5. Готовит сегментированные очереди для рассылки.

Человек (партнер):
1. Проверяет топ-10/топ-20 лидов в день.
2. Подтверждает релевантность и отправляет первые касания.
3. Закрывает ответы в звонки/сделки.

Итог: найм не нужен для MVP. Можно работать текущими силами.

## 3) Гео-структура (раздельные базы)
1. `RU_CIS` — отдельный pipeline, отдельные шаблоны писем (RU).
2. `CYPRUS_EN` — отдельный pipeline пилота, отдельные шаблоны (EN/GR).
3. На уровне карточки обязательные поля: `geo_bucket`, `source`, `language`, `deadline`, `budget`, `cpv`, `score`, `status`.

## 4) Техническая архитектура в текущем проекте
1. Новый локальный сервис сбора лидов в `helper/`:
   - `helper/lead_collector_server.js`
   - хранение кэша лидов в `helper/.cache/lead_cache_v1.json`
2. Новый модуль коннекторов:
   - `helper/sources/cyprus_eprocurement.js`
   - `helper/sources/ted.js`
   - `helper/sources/ru_cis_*.js` (по whitelist источников)
3. Единый формат лида (JSON) + дедупликация по `source_id + buyer + title + deadline`.
4. Добавление в UI:
   - страница/блок `Автосбор`
   - кнопки `Проверить источник`, `Запустить сбор`, `Импорт в карточки`
5. Автозапуск:
   - Windows: Task Scheduler (каждые 2-4 часа)
   - Mac: launchd/cron (каждые 2-4 часа)

## 5) Этапы реализации

## Этап A (MVP, 5-7 дней)
1. Каркас `lead_collector_server`.
2. Коннектор `CYPRUS_EN`: eProcurement (пилот).
3. Коннектор `TED` по фильтрам.
4. Нормализация карточки + дедуп + импорт в текущую БД.
5. Простой scoring:
   - бюджет (вес высокий),
   - близость дедлайна (средний),
   - совпадение CPV/ключевых слов (высокий),
   - гео/язык (средний).
6. Результат: автообновление базы Cyprus + EU.

## Этап B (RU/CIS, 4-7 дней)
1. Подключение утвержденных источников RU/CIS (после whitelist).
2. Раздельные правила scoring для RU/CIS.
3. Раздельные очереди и шаблоны рассылок RU/EN.
4. Результат: две независимые ленты лидов.

## Этап C (CRM-полуавтомат, 3-5 дней)
1. Автозадачи follow-up (D+3, D+7, D+14).
2. Журнал причин отказа.
3. A/B темы писем по сегментам.
4. Результат: меньше ручной рутины и лучше конверсия в ответ.

## Этап D (аналитика, 2-3 дня)
1. Воронка: `лиды -> отправки -> ответы -> звонки -> сделки`.
2. Конверсия по гео/шаблону/источнику.
3. Топ-источники по ROI.

1. Автособранных новых лидов: 300+.
2. Доля релевантных после ручной проверки: 30%+.

## 8) Риски и контроль
1. Ограничения по парсингу источников: используем только легальные/доступные методы и официальные точки входа.
2. Перегрузка нерелевантными лидами: настраиваем фильтры и scoring итеративно раз в неделю.
3. Риски по рассылкам (EU): соблюдаем opt-out и ведем suppression list.
